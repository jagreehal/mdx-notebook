# Build an AI agent with tool use using Vercel AI SDK and Gemini

The Vercel AI SDK gives you a clean abstraction for building AI agents that call tools to answer questions. This tutorial walks through building one from zero with Google's Gemini 2.5 Flash, capturing every tool call at build time so the documentation you ship matches what the code actually does.

If your AI SDK calls have ever silently changed behavior between commits, this approach catches it: the tool-call sequence is part of the build output, and a diff in your rendered page shows up the moment the LLM's behavior shifts.

## What you'll build

A travel-assistant agent. Ask it "should I pack a jacket for Tokyo?" and it calls a `getWeather` tool with the right city, reads the response, and writes a useful reply. By the end you'll have two tools, a `<MessageThread>` showing the conversation, and a `<ToolCallTimeline>` showing every invocation with its input, output, and latency.

## Prerequisites

- Node 20 or higher
- A Google AI API key from [aistudio.google.com/apikey](https://aistudio.google.com/apikey) (Gemini 2.5 Flash has a generous free tier)
- Basic TypeScript

Be specific in the description; vague tools confuse the model.

## Step 2 — Wire it into `generateText`

The AI SDK's `generateText` takes a `tools` map and a `maxSteps` cap. When the model calls a tool, the SDK runs it, feeds the result back into the next step, and continues until the model produces text without calling another tool or hits `maxSteps`.

```ts
import { generateText, tool } from "ai";
import { createGoogleGenerativeAI } from "@ai-sdk/google";

const google = createGoogleGenerativeAI({ apiKey: process.env.GOOGLE_API_KEY });

const result = await generateText({
  model: google("gemini-2.5-flash"),
  tools: { getWeather },
  maxSteps: 5,
  messages: [{ role: "user", content: "What's the weather in Tokyo?" }]
});
```

`maxSteps` is your safety cap — without it, a buggy agent that keeps calling tools could spin forever. Five is a reasonable default for most use cases.

## What you got

A working AI agent with two tools, every invocation captured at build time, and a rendered visualization that updates automatically when the model's behavior changes. You can ship this as documentation that genuinely reflects what runs.

Where to go next:

- Add a third tool — see how Gemini handles ambiguous queries.
- Replace mocked tools with real APIs (e.g., a weather provider, a database query). The pattern is identical.
- Stream the response with the AI SDK's `streamText` and capture it via the [streaming tutorial](/tutorials/03-streaming).
- Use the [run matrix](/tutorials/05-crash-resume) to try the same prompt across model temperatures or against different providers.

The Vercel AI SDK docs at [sdk.vercel.ai/docs](https://sdk.vercel.ai/docs) cover everything else (streaming, structured output, embeddings). What this tutorial gave you was the *captured render* discipline — your docs and your agent stay in sync because they're literally the same artifact.
